The navigation subsystem uses the Nav2 stack for autonomous path planning, trajectory tracking, and collision avoidance. After receiving a goal pose or waypoint sequence, the rover performs autonomous motion as a closed-loop pipeline: sensor acquisition, costmap update, global path planning, local trajectory generation, and velocity command execution. The same stack serves traversal, sampling/maintenance positioning, and task-specific goal poses, while manipulation details are delegated to dedicated subsystems (\autoref{par:arm_kinematics_motion_planning}).

Global planning uses NavFn (Dijkstra) over a \qty{0.05}{\metre} resolution 2D costmap, while local motion is generated by the DWB controller tracking the global path and reacting online to terrain changes and newly detected obstacles. Collision avoidance with other rovers is enforced through obstacle layer detection and safety-margin inflation.

The costmap integrates stereo depth, ToF range data, and LiDAR scans into voxel, obstacle, and inflation layers at \qty{0.05}{\metre} resolution. An obstacle is any non-traversable region: occupied cells from depth/LiDAR, terrain protrusions above \qty{0.15}{\metre}, depressions deeper than \qty{0.10}{\metre}, or cells with cost $\geq 253$ after inflation. Terrain traversability is estimated by slope and roughness, with traversable cells at cost 0--252 and impassable cells at cost $\geq 253$. Collision avoidance operates at two levels: NavFn plans only through traversable cells, while DWB rejects trajectories intersecting impassable obstacles.

If the path becomes obstructed, the navigation state machine (Figure~\ref{fig:behavior_sm}) executes: local evasion attempt, global replanning if an alternative path exists, or fail-safe stop if no safe trajectory is found within a defined timeout.
